# 获取日期时间技能 (get-date) v2.0

基于 skill-creator-pro 最佳实践优化的日期时间查询技能。

这是一个按照 Operator 范式设计的日期时间查询技能，用于可靠地获取当前系统日期和时间信息。技能采用脚本优先的设计理念，确保每次调用都能准确返回格式化的时间信息。

## 使用场景

### 应该触发的情况
- ✅ "现在几点了？" → 返回当前时间
- ✅ "今天星期几？" → 返回今天是星期几
- ✅ "今天的日期是什么？" → 返回完整日期
- ✅ "告诉我完整的日期时间" → 返回完整日期时间信息
- ✅ "现在是几月？" → 返回当前月份
- ✅ "今年是哪一年？" → 返回当前年份

### 不应该触发的情况
- ❌ "2023年3月19日是星期几？"（历史日期）
- ❌ "纽约现在几点了？"（时区转换）
- ❌ "100天后是几月几号？"（日期计算）
- ❌ "这个月有多少天？"（日历查询）

4. 返回自然语言回复

## 设计原则

### Operator 范式
本技能采用 Operator（操作员）范式设计，核心特点是：
- **脚本优先**：所有操作都封装在脚本中
- **确定性执行**：相同输入产生相同输出
- **严格验证**：执行前后都有验证检查
- **错误处理**：完整的失败检测和恢复策略

### 边界优先设计
在创建技能前明确定义了四个边界问题：
1. **解决什么问题**：频繁查询当前日期时间
2. **什么触发**：日期时间相关查询
3. **什么不触发**：历史日期、时区转换、日期计算
4. **成功标准**：准确返回格式化的系统时间

## 技术实现

### 核心脚本
`scripts/get_datetime.py` 包含三个核心函数：
1. `get_current_datetime()`：获取系统时间并格式化
2. `detect_intent()`：分析用户查询意图
3. `format_by_intent()`：根据意图格式化输出

### 意图识别
基于关键词匹配的意图检测系统，支持6种意图分类：
- time（时间查询）
- weekday（星期查询）
- date（日期查询）
- full（完整信息查询）
- month（月份查询）
- year（年份查询）

### 输出模板
可配置的输出模板系统，支持：
- 中英文自适应
- 表情符号美化
- 错误处理模板
- 自定义格式扩展
