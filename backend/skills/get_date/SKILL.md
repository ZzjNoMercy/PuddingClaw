# 获取日期时间技能

## Goal
解决用户频繁查询当前日期和时间信息的重复问题。提供准确、格式化的系统时间信息，支持多种查询意图（时间、日期、星期、月份、年份等）。确保每次调用都能可靠地返回正确的系统时间。

## Workflow
1. **输入解析**：分析用户问题，识别查询意图（时间、日期、星期、月份、年份、完整信息）
2. **时间获取**：执行 `scripts/get_datetime.py` 获取当前系统时间
3. **格式转换**：根据查询意图选择相应的输出格式
4. **结果验证**：验证时间获取是否成功，格式是否正确
5. **输出生成**：生成自然语言回复，包含格式化的日期时间信息

## Decision Tree
- 如果用户询问"几点了"、"时间"、"现在几点" → 返回当前时间 (HH:MM:SS)
- 如果用户询问"星期几"、"周几" → 返回今天是星期几
- 如果用户询问"日期"、"几月几号"、"几号" → 返回完整日期 (YYYY年MM月DD日 星期X)
- 如果用户询问"完整"、"全部" → 返回完整日期时间信息
- 如果用户询问"几月"（不包含"年"） → 返回当前月份
- 如果用户询问"哪一年"、"今年" → 返回当前年份
- 如果用户询问其他时间相关问题 → 返回完整日期时间信息

## Constraints
- 只查询当前系统时间，不处理历史日期或未来日期
- 不处理时区转换，使用系统本地时间
- 不进行日期计算（如"100天后"）
- 输入必须是当前时间相关的查询
- 如果系统时间获取失败，返回错误信息并建议用户检查系统时间设置

## Validation
- **前置验证**：检查用户问题是否包含时间相关关键词
- **执行验证**：验证 `datetime.now()` 调用是否成功
- **输出验证**：检查生成的日期时间字符串是否符合预期格式
- **失败处理**：
  - 如果时间获取失败，返回"无法获取当前时间，请检查系统时间设置"
  - 如果格式转换失败，返回默认格式的时间信息
  - 如果意图识别失败，返回完整的日期时间信息
