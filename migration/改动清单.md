## 6. `backend/graph/middlewares/compression.py` — 局部替换

替换 `count_tokens_tiktoken` 函数，增加计入 `tool_calls` token：

```python
def count_tokens_tiktoken(messages) -> int:
    total = 0
    for m in messages:
        content = m.content if hasattr(m, "content") else str(m)
        if isinstance(content, list):
            content = "".join(block.get("text", "") if isinstance(block, dict) else str(block) for block in content)
        if not isinstance(content, str):
            content = str(content)
        total += _encode_text(content)
        # 新增：计入 tool_calls
        if hasattr(m, "tool_calls") and m.tool_calls:
            for tc in m.tool_calls:
                tc_text = json.dumps(tc, ensure_ascii=False)
                total += _encode_text(tc_text)
    return total
```

并在同一文件新增：
```python
def count_text_tokens(text: str) -> int:

```

## 10. Token 用量统计功能 — 新增

> 参考 `migration/token_usage_tracking.md`，适配当前项目。用 **SQLite + jsonl 双写**，零额外依赖，后续如需切 PostgreSQL 只需改连接字符串。
>
> **数据库选型说明**：当前项目只有 Token 统计需要结构化存储，PostgreSQL 有点重。SQLite 足够胜任，后续统一迁移成本低。

### 10.1 存储层：`backend/graph/token_usage_store.py` — 新建

**职责**：每轮 LLM 调用后双写 token 用量到 SQLite + jsonl。

**SQLite 表结构**：
```sql
CREATE TABLE token_usage (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id TEXT,
    session_id TEXT,
    round_num INTEGER,
    input_tokens INTEGER,
    output_tokens INTEGER,
    total_tokens INTEGER,
    start_time REAL,
    timestamp REAL
);
```

**关键行为**：
- 数据库文件：`backend/data/token_usage.db`（自动创建）
- jsonl 文件：`backend/data/stats/tokens/YYYY-MM-DD.jsonl`
- 写入失败静默跳过，不阻塞主流程

### 10.2 Agent 核心：`backend/graph/agent.py` — 修改

1. **LLM 初始化增加 `stream_usage=True`**（`ChatDeepSeek` 构造参数）
2. **`_run_agent_stream` 捕获 `usage_metadata`**：
   - 在 `updates` 模式、`node_name == "model"` 处读取 `agent_msg.usage_metadata`
   - 提取 `input_tokens` / `output_tokens`，fallback 到 tiktoken 估算
   - 调用 `token_usage_store.record(...)` 写入

**记录时机**：
```
用户提问 → LLM Stream → usage_metadata（最后一个 chunk）
         → node_name="model" 更新
         → 读取 usage_metadata
         → record_token_usage() 双写 SQLite + jsonl
         → （如果有 tool_calls）→ 工具执行 → 下一轮 model
```

**数据结构**：
```python
{
    "user_id": "default_user",
    "session_id": "session-xxx",
    "round": 1,
    "input_tokens": 8000,
    "output_tokens": 1200,
    "total_tokens": 9200,
    "start_time": 1780993283.12,
    "timestamp": 1780993285.45
}
```
